---
name: Design Patterns
description: Padrões de projeto mais comuns e quando aplicar cada um
---

| Pattern | Quando usar |
|---|---|
| **Strategy** | Comportamento variável que precisa ser trocado em runtime sem alterar o código cliente |
| **Repository** | Abstrair acesso a dados; o serviço não sabe qual banco ou storage está por baixo |
| **Factory** | Centralizar criação de objetos cuja implementação concreta depende de configuração |
| **Adapter** | Integrar dependência externa com interface incompatível sem contaminar o domínio |
| **Observer** | Desacoplar produtor de evento dos seus consumidores; múltiplos handlers independentes |
