---
name: OOP — fundamentos
description: Encapsulamento, abstração, polimorfismo e herança
---

- **Encapsulamento**: estado interno privado; API pública mínima e intencional
- **Abstração**: definir contratos (interfaces) antes de implementações — o "o quê" antes do "como"
- **Polimorfismo**: depender de interfaces; qualquer implementação que respeite o contrato é válida
- **Herança**: usar com cautela. Preferir composição. Herança só para relações genuínas de "é um"
