---
name: SOLID
description: Os 5 princípios SOLID de design orientado a objetos
---

| Princípio | Regra |
|---|---|
| **SRP** — responsabilidade única | Um módulo, um motivo para mudar. Se faz coisas demais, extrair. |
| **OCP** — aberto/fechado | Estender comportamento via novas implementações, não editando código existente. |
| **LSP** — substituição de Liskov | Implementações de uma interface devem ser intercambiáveis sem quebrar o sistema. |
| **ISP** — segregação de interface | Interfaces pequenas e focadas. Não forçar dependência de métodos não usados. |
| **DIP** — inversão de dependência | Depender de abstrações, não de implementações concretas. |
