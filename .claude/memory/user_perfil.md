Davi é o desenvolvedor e arquiteto deste projeto. Trabalha em múltiplos PCs, por isso tudo que precisa persistir entre sessões deve ficar no repositório git — não em `~/.claude/`.

Prefere respostas diretas e objetivas. Não precisa de explicações longas quando o ponto é simples.

Este é o primeiro projeto onde está seguindo uma metodologia estruturada de desenvolvimento com Claude Code (baseada no [docs/FRAMEWORK_PESSOAL.md](../docs/FRAMEWORK_PESSOAL.md)).

Orientado a processo e completude — cobra quando o fluxo não é seguido até o fim (memórias não atualizadas, checklists não marcados, commits sem padrão). Espera que cada iteração feche o ciclo completo.

Visão de longo prazo: usar Claude como orquestrador e agentes mais baratos (ex: Codex) como executores de código — economizando tokens sem perder qualidade. Isso requer AGENTS.md como contrato entre agentes.
